Summary of produced files
=========================

Context
-------
Project: Avalonia UI 11.3.x / .NET 10, Fluent theme, Inter font, compiled bindings enabled.
Request: Dark-theme enterprise ERP login screen with email field, password field,
         login button and "forgot password" link.

Files produced
--------------

1. LoginView.axaml
   Main view — a UserControl with a two-column Grid layout:
   - Left column (flex width): dark branding panel with a logo placeholder,
     product name "NexusERP", tagline and a brief descriptor.
   - Right column (fixed 480 px): the actual login form card containing:
       * Form heading ("Bem-vindo de volta") and subtitle
       * "E-mail corporativo" TextBox (Classes="LoginField", watermark: nome@empresa.com.br)
       * "Senha" TextBox (Classes="LoginField", PasswordChar bullet, watermark: ••••••••)
       * "Esqueci minha senha" link Button (Classes="LinkButton"), right-aligned
       * "Entrar" primary Button (Classes="PrimaryButton", full-width)
       * Footer legal note
   The view references /Styles/LoginStyles.axaml via <StyleInclude>.
   TabIndex order: email → password → forgot link → login button.

2. LoginStyles.axaml
   Standalone <Styles> file (intended to live at /Styles/LoginStyles.axaml in the project,
   or included directly in App.axaml).
   Contains:
   - Design tokens as StaticResources (Color + SolidColorBrush pairs):
       ColorBackground #0F1117, ColorSurface #13161F, ColorSurfaceRaised #1A1E2A,
       ColorBorder #232840, ColorBorderFocused #4F8EF7,
       ColorTextPrimary #E8ECF4, ColorTextSecondary #5A6478, ColorTextMuted #3E4A60,
       ColorAccent #4F8EF7, ColorAccentHover #6BA3FA, ColorAccentPressed #3A75E0.
   - Style classes:
       FieldLabel   — 12 px medium label above each field
       LoginField   — custom TextBox template: dark raised surface, 1 px border,
                      8 px corner radius, focus highlight (accent border + slightly
                      lighter background), hover highlight, watermark hidden on
                      focus/non-empty
       PrimaryButton — full-width accent-filled button (46 px height), hover/pressed
                       states, 150 ms background transition
       LinkButton   — borderless transparent button, accent foreground, underline on
                      hover, 100 ms transition

3. LoginView.axaml.cs
   Minimal code-behind partial class for LoginView (no logic — ready for ViewModel
   binding via x:DataType when a LoginViewModel is added).

Design decisions
----------------
- Split-panel layout: common enterprise ERP pattern; branding on the left keeps the
  form uncluttered while reinforcing brand identity.
- All colours are inline hex values in the token resources so the file is self-contained
  and works without a separate app-level resource dictionary.
- No hard-coded colours appear directly in the view markup — all colours go through the
  named brushes in LoginStyles.axaml, making future rebranding straightforward.
- TabIndex is explicitly set for keyboard-only navigation compliance.
- The password field uses PasswordChar with a bullet character; to add a show/hide
  toggle, bind PasswordChar to a ViewModel property toggled by an overlay Button.
- ScrollViewer wraps the form so it remains usable on small/portrait windows.
- Compiled bindings (AvaloniaUseCompiledBindingsByDefault=true) are respected:
  no bindings are present in the view; add x:DataType="vm:LoginViewModel" when
  wiring up a ViewModel.

Integration notes
-----------------
- Place LoginStyles.axaml in the project under a /Styles/ folder and add:
      <StyleInclude Source="/Styles/LoginStyles.axaml" />
  to App.axaml (inside <Application.Styles>), OR keep the <UserControl.Styles>
  <StyleInclude> already present in LoginView.axaml for self-contained styling.
- Register the view in navigation or set it as the main content of MainWindow.
